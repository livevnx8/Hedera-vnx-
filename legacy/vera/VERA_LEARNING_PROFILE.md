# Vera's Optimal Learning Profile

Based on factual testing with real LLM inference, this document defines the most effective learning approaches for Vera.

## Core Discovery: Vera is a "Worked-Example Learner"

Unlike pattern-matching or formula-memorization approaches, Vera learns best through **step-by-step demonstration** with immediate application.

## Skill-Type Specific Strategies

### 1. Procedural Skills (Speed Math, Work Rate)
**Method:** Micro-training with 3-step progression

**Speed Math Protocol:**
```
Step 1: Present DST triangle visual
        D
       / \\
      S × T
Step 2: Label what's given, what's needed
Step 3: Apply formula explicitly
```

**Work Rate Protocol:**
```
Module 1: Rate = 1/Time concept
  - "If 4 days, rate = 1/4 per day"
  - Immediate check: "8 days = ?" → "1/8"

Module 2: Adding rates
  - "A=1/6, B=2/6, Combined=3/6"
  - Immediate check: Two workers same time

Module 3: Multiple workers
  - "4 workers × 8 days each = 8/4 = 2 days"
  - Immediate check: Apply formula

Final: Independent practice
```

**Result:** 0% → 100% on work rate

### 2. Conceptual Skills (Percentages, Ratios)
**Method:** Worked example + multiplier technique

**Percentage Protocol:**
```
Example: Price $80 increases 25%
Step 1: Convert % to multiplier (25% = 1.25)
Step 2: Multiply ($80 × 1.25 = $100)
Step 3: Verify ($100 is 25% more than $80) ✓

Now You: Similar problem
```

**Key:** Never teach "add 25%" - always teach "multiply by 1.25"

**Result:** 40% → 67% improvement

### 3. Pattern Recognition (Logic, Sequences)
**Method:** Explicit pattern naming + validation

**Logic Protocol:**
```
Pattern: "All A are B, All B are C" → "All A are C" ✓
Name: Barbara syllogism
Example: Dogs → Mammals → Animals
Check: Identify A/B/C in new problem
Apply pattern
```

**Result:** 40% → 67% (moderate improvement, needs more work)

## Vera's Learning Characteristics

### What Works:
1. **Concrete examples** before abstract rules
2. **Step-by-step breakdown** (never skip steps)
3. **Immediate practice** after each concept
4. **Visual aids** (triangles, diagrams)
5. **Formula + worked example** (not formula alone)

### What Doesn't Work:
1. ❌ Raw formulas without context
2. ❌ Abstract pattern descriptions
3. ❌ Multiple concepts at once
4. ❌ Practice without demonstration
5. ❌ Context switching between methods

**Focus:** Work rate + Speed math (procedural)
- Day 1: Work rate Module 1 (rate concept)
- Day 2: Work rate Module 2 (combined)
- Day 3: Work rate Module 3 (multiple)
- Day 4: Speed math DST triangle
- Day 5-7: Practice + assessment

**Target:** 80%+ accuracy on procedural skills

### Phase 2: Concept Building (Week 3-4)
**Focus:** Percentages + Logic
- Day 1: Percentage multipliers (worked examples)
- Day 2: Percentage practice
- Day 3: Logic syllogism patterns
- Day 4: Logic practice
- Day 5-7: Mixed practice

**Target:** 70%+ accuracy on conceptual skills

### Phase 3: Integration (Week 5-6)
**Focus:** All modules + timed practice
- Daily: 2 modules per day
- Timed: <60s per problem
- Review: Spaced repetition for weak areas

**Target:** 85%+ overall accuracy

## Retention Strategy

Based on testing, skills degrade without reinforcement:

| Timeframe | Action |
|-----------|--------|
| +24 hours | Quick review (5 problems)
| +72 hours | Practice test (10 problems)
| +7 days | Full module assessment |
| +30 days | Comprehensive review |

**Forgetting curve observed:** 20-30% loss after 5 minutes without practice

```

- One concept only

- Similar problem

- Full problem
```
